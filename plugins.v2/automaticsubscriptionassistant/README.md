# 自动订阅助手（AutomaticSubscriptionAssistant）

统一聚合 **豆瓣榜单 / 猫眼榜单 / 热门媒体 / Mikan 季度新番 / 奈飞榜单** 五类来源，按你的规则组合过滤后**自动订阅**的 MoviePilot 插件。每个来源可单独开关、单独定时、单独手动运行，所有来源共用一套「识别 → 查重 → 订阅 → 记历史」落地管线，绝不重复订阅。

## 功能特性

- **五源聚合**：豆瓣榜单、猫眼榜单、热门媒体、Mikan 季度新番、奈飞 Top10，一个插件全覆盖。
- **每源独立调度**：每个来源有独立的启用开关与定时（cron），可分别开关、分别手动运行。
- **组合过滤**：按发行年份、评分、媒体类型、订阅人次等条件筛选后再订阅。
- **绝不重复订阅**：媒体库查重 + 订阅查重 + 跨来源去重三重保险。
- **订阅管理**：对本插件创建的订阅一键退订 / 暂停 / 恢复。
- **订阅历史**：完整记录每次处理结果（已订阅 / 已存在 / 已过滤 / 异常等），支持多条件筛选；异常项可**重新识别**。
- **连通性测试**：一键验证任意来源能否取到数据，**不产生真实订阅**，带防抖防止误触。
- **状态检测**：保存时自动关闭「开了却没设置任何监听内容」的来源，避免空转。
- **多语言界面**：跟随 MoviePilot 语言，支持简体中文 / 繁體中文 / English。
- **桌面与移动端自适应**：配置、历史、管理页均适配手机竖屏。

## 安装

1. 在 MoviePilot 的「插件市场」中安装「自动订阅助手」。
2. 安装后在「插件」页找到它，点击进入配置。

> ### ⚠️ 前端需构建产物（自建 / 开发时注意）
>
> 本插件配置页与历史页是 **Vue3 联邦模块**，依赖 `frontend/dist/assets` 构建产物。**通过插件市场安装的发行版已包含构建产物，无需手动构建。**
>
> 仅当你从源码自行部署且 `frontend/dist` 缺失时，需要手动构建：
>
> ```bash
> cd frontend
> yarn install   # 首次
> yarn build     # 生成 frontend/dist/assets/*
> ```
>
> - **未构建**：配置页 / 历史页打不开（宿主加载联邦模块失败）。
> - **但**：后端定时任务、HTTP API 与前端完全独立——即使前端没构建，定时订阅与手动触发仍能正常运行。

## 快速开始（3 步）

1. **打开总开关**：进入配置 →「全局设置」→ 打开「启用插件」。
2. **配置一个来源**：在左侧选一个来源（如「豆瓣榜单」）→ 打开「启用该来源」→ 选好要监听的榜单/内容 → 设定定时规则。
3. **保存**：点「保存」。到点后自动运行；也可在该来源页点「立即运行一次」马上跑一遍，或先点「连通性测试」确认能取到数据。

订阅结果可在顶部「订阅历史」查看，已创建的订阅可在「订阅管理」里退订 / 暂停。

## 使用方法

配置页顶部有三个标签页：**订阅配置**、**订阅历史**、**订阅管理**。下面按使用顺序逐一说明。

### 全局设置

「订阅配置」→ 左侧「全局设置」，作用于所有来源：

| 设置 | 说明 |
|------|------|
| **启用插件** | 总开关，关闭后所有定时任务停止（各来源单独的开关不再生效）。 |
| **运行通知** | 每次运行后是否发送消息通知。 |
| **订阅用户名** | 新建订阅记录归属的用户名，默认「自动订阅助手」。 |
| **媒体库已存在仍订阅** | 媒体库里已有资源时是否仍然添加订阅。 |
| **保存后立即运行一次** | 保存后立刻把所有已启用来源跑一遍，随后自动复位。 |
| **清空历史记录** | 保存后清空全部订阅历史，随后自动复位。 |

### 来源配置总览

每个来源都有一致的结构：

- **启用该来源**：本来源的开关（受全局总开关约束）。
- **定时规则**：五位 cron 表达式，留空则用该来源的默认值。
- **抓取选项**：决定「监听什么内容」（各来源不同，见下）。
- **过滤条件**：决定「订阅哪些」（年份 / 评分 / 类型等）。
- **高级选项**：不常改的项默认折叠隐藏。
- 底部有 **立即运行一次** 与 **连通性测试** 两个按钮。

> **提示**：留空即「不过滤 / 全部」。多数来源开箱即用，选好要监听的榜单再保存即可。

### 各来源详解

#### 🎬 豆瓣榜单（默认定时 `0 8 * * *`）

通过 RSSHub 抓取豆瓣榜单。

- **榜单**：多选内置榜单（热门电影、热门电视剧、TOP250 等）。留空且无自定义 RSS = 不监听。
- **RSSHub 基址**（高级）：默认 `https://rsshub.app`。若该地址在你的网络被墙，可改为自建 RSSHub 实例（如 `https://rsshub.你的域名`）。
- **自定义 RSS 地址**（高级）：每行一个完整 RSS 地址，可对接任意源，不受基址影响。
- **使用代理**：抓取是否走代理。

#### 🎟️ 猫眼榜单（默认定时 `0 9 * * *`）

抓取猫眼票房榜与网播热度榜。

- **类型**：电影票房、网播热度、网播剧集、综艺等。
- **平台**：网播类可细分平台（全网 / 腾讯 / 爱奇艺 / 芒果 / 优酷 / 搜狐 / 乐视 / PPTV）。
- **每榜条数**：每个榜取前 N 条。

> ⚠️ 网络电影因数据源已停更，暂不支持。

#### 🔥 热门媒体（默认定时 `5 1 * * *`）

读取 **MoviePilot 官方服务器的「热门订阅统计」**（大家都在订阅的热门内容），分「电影订阅 / 剧集订阅」两组，各自独立配置：

- **电影订阅 / 剧集订阅**：两类各一个开关（默认开）。
- **风格**：TMDB 风格标签多选，留空=该类型全部。
- **获取条数 / 评分下限 / 订阅人次**：各类型独立设置，`0` = 不限。

> ### 🔑 必须开启「订阅数据共享」
>
> 热门媒体依赖 MoviePilot 全局的 **「订阅数据共享」（`SUBSCRIBE_STATISTIC_SHARE`）**：未开启时官方服务端直接返回空，本来源**取不到任何数据**。
>
> 本插件在热门媒体配置区提供了一个 **「订阅数据共享」开关**，打开它即可**一键开启并持久化**主程序的该设置（等同于在 MoviePilot『设定 → 订阅』里开启，即时生效、无需重启）。配置区顶部也有黄色公告提示。
>
> 若该开关打不开（提示被系统环境变量锁定），说明你用 Docker `environment:` 或 `export` 显式设置了该变量，请到那里改。

#### 📺 Mikan 季度新番（默认定时 `0 10 * * 1`）

抓取蜜柑计划（mikanani.me）的季度番剧列表。

- **年份**：`0` = 当前年。
- **季度**：`当前`（按月份自动推导）或 春 / 夏 / 秋 / 冬。
- **解析 bangumi id**（默认开）：逐条抓详情页补 bgm.tv 编号，识别更准但更慢（每条约 0.6 秒）；关闭则仅按标题 + 年份识别。

> Mikan 始终抓取当前季度、没有「监听内容」开关，因此**只要启用就一定会尝试抓取**（不会被状态检测自动关闭）。

#### 🍿 奈飞榜单（默认定时 `0 11 * * 3`）

抓取 Netflix 官方 Tudum Top10 公开榜单。**全球榜**与**国家榜**相互独立、可同时启用。

- **全球榜**（默认开）+ **全球媒体类型**（默认全选 4 类：英语/非英语 × 电影/剧集）。
- **全球数据源**：最新周榜（默认）或「史上最热（不分周）」。
- **国家/地区 × 媒体类型**：按地区分别选择要监听的媒体类型（弹窗式卡片管理，可对每个地区单独开关而不删除配置）。
- **每榜取前 N**：默认 10。
- **使用代理**（默认关）：**若你的服务器无法直连 `netflix.com`，务必开启并在 MoviePilot 配好代理**，否则取不到数据。
- **富元数据模式**（高级，默认关）：改抓榜单页内嵌数据，带年份 + 干净剧名，识别更准；非英语两类无此富页会回退普通模式。
- **周更缓存**（高级，默认开）：奈飞每周更新一次，开启缓存避免同一周重复抓取触发风控。

> **全球榜或国家榜都要选中「媒体类型」才会抓取**——两者都为空时本来源不监听任何内容（会被状态检测自动关闭）。

### 过滤条件

部分来源支持在「过滤条件」区进一步筛选（留空/`0` 表示不启用该条件）：

- **发行年份 ≥**：只订阅不早于某年的内容。
- **评分 ≥**：只订阅评分达标的内容。
- **媒体类型**：只订阅电影 / 只订阅电视剧 / 全部。

### 连通性测试

每个来源配置区底部有 **「连通性测试」** 按钮：只抓取前几条验证**能否取到数据**，**绝不产生真实订阅、绝不写入历史**。用来在正式启用前确认来源可达、配置有效。

- 带**防抖**：进行中禁用 + 冷却间隔，防止短时间高频点击。
- 结果会明确提示「连通正常，取到示例：xxx」或失败原因。

### 状态检测（自动关闭无监听来源）

**保存时**，插件会检查每个已启用的来源：如果它**没有设置任何要监听的内容**（启用了也不会有产出），就**自动关闭该来源开关**并提示，避免无意义的空转。判定依据各来源不同，例如：

- 豆瓣：没选任何榜单且没有自定义 RSS；
- 猫眼：没开票房榜且没选任何网播平台/类型；
- 奈飞：全球榜和国家榜都没选媒体类型；
- 热门媒体：电影/剧集都关了，**或**「订阅数据共享」没开；
- Mikan：始终有监听，不参与自动关闭。

### 订阅管理

顶部「订阅管理」标签页列出**本插件创建的订阅**。每张卡片右下角的 **「更多」** 菜单里可以：

- **退订**：删除该订阅。
- **暂停 / 恢复**：临时停用 / 重新启用该订阅。

支持多选批量操作，以及按名称搜索、按年份/类型/状态筛选（筛选为弹窗式）。

### 订阅历史

顶部「订阅历史」标签页记录每次运行的处理结果，状态包括：已订阅、媒体库已存在、已有订阅、已过滤、异常等。

- **筛选/搜索**（弹窗式）：按关键词、发行年份区间、类型、状态、来源多条件过滤。
- **重新识别**：对「异常 / 未识别」的记录（如未上映、季度预告等暂无 TMDB/Bangumi 记录的条目），可在「更多」菜单点 **重新识别** 手动重跑一次识别与订阅。

## 常见问题 FAQ

### ❓ 热门媒体一直取不到数据

按顺序检查：

1. **是否开启了「订阅数据共享」**——这是最常见原因。在热门媒体配置区打开「订阅数据共享」开关（或在 MoviePilot『设定 → 订阅』开启 `SUBSCRIBE_STATISTIC_SHARE`）。未开启时服务端直接返回空。
2. 若开启后仍为空，看运行日志：新版日志会明确区分「服务端返回 200 但无数据」「连不上/超时」「HTTP 异常」。
3. 若你配置了「风格」筛选却为空，试试**清空风格改用全部**（服务端对个别风格可能暂无数据）。

> 热门媒体展示的就是主程序「热门订阅」的同一份数据；插件已与官方前端使用一致的请求方式。

### ❓ 奈飞榜单取不到数据

1. **网络**：奈飞数据来自 `netflix.com`，若你的服务器（尤其国内）无法直连，请**开启奈飞的「使用代理」开关**并在 MoviePilot 配好代理。
2. **媒体类型**：确认「全球媒体类型」或「国家 × 媒体类型」至少选了一项，否则不会抓取。
3. 数据源本身正常时，连通性测试应能取到示例。

### ❓ 配置页 / 历史页打不开（空白）

前端联邦模块未构建或加载失败。发行版已内置构建产物；自建时请在 `frontend/` 下 `yarn build`（见[安装](#安装)）。后端定时订阅不受影响。

### ❓ 连通性测试显示「已连通但未取到条目」

表示请求成功但该来源当前无匹配数据，常见于：过滤条件过严、该来源当前确无内容、或热门媒体未开「订阅数据共享」。放宽过滤或按上面对应来源的排查处理。

## 开发者参考

### 后端 API

所有接口挂在 `plugin/AutomaticSubscriptionAssistant/` 下，读取类 `auth="bear"`，删除类可用 `auth="apikey"`。

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/providers` | 列出所有来源的 `ProviderSpec`，供前端动态渲染 |
| GET / POST | `/config` | 读取 / 保存完整配置（保存后重建调度） |
| GET | `/status` | 各来源启用状态与累计统计 |
| POST | `/run` | 手动触发指定来源立即运行一次（3 秒后） |
| POST | `/providers/test` | 连通性测试（不订阅、不写历史，带最小间隔防抖） |
| GET | `/history` | 查询历史，支持来源/状态/类型/关键词/年份区间过滤与分页 |
| DELETE / POST | `/history` · `/history/delete` | 按 unique 删除一条历史 |
| POST | `/history/batch-delete` | 批量删除历史 |
| POST | `/history/recognize` | 对一条历史重新识别并尝试订阅 |
| GET | `/subscribes` | 列出本插件创建的订阅 |
| POST | `/subscribes/delete` | 批量退订 |
| POST | `/subscribes/state` | 批量暂停 / 恢复 |
| GET / POST | `/system/share` | 读取 / 设置 MoviePilot 全局「订阅数据共享」开关 |

### 新增一个订阅源

分层设计下，新增来源只需实现抓取与解析，落地 / 过滤 / 历史 / 前端配置块全部复用：

1. 在 `providers/xxx.py` 写一个 `RankProvider` 子类，实现 `get_spec()`（声明配置项与默认 cron）和 `fetch()`（抓取解析为 `RankMediaItem`），用 `@register` 装饰。
2. 在 `providers/__init__.py` 追加 `from . import xxx` 触发注册。
3. 完成——前端配置页、定时调度、落地管线、去重、历史全部自动复用。

新增**过滤条件**同理：在 `core/filters.py` 加一个 `MediaFilter` 子类并注册进 `BUILTIN_FILTERS`，再在某来源的 `filters_schema` 里声明启用。
